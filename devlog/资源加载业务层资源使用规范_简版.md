# 资源加载业务层资源使用规范_简版

## 适用范围

适用于客户端业务层使用当前 `AssetBundle` 资源框架进行资源加载、场景加载、预热和释放的所有场景。

## 核心原则

1. 正式环境统一使用异步加载接口。
2. 业务层可以放弃结果，但不打断底层共享加载。
3. 谁发起加载，谁负责释放。
4. 资源对象不是生命周期入口，`AssetHandle<T>` 才是。
5. 预热引用和业务引用分开计算，必须分别释放。

## 必须遵守

### 1. 资源加载后必须保存 `AssetHandle<T>`

正确：

```csharp
private AssetHandle<GameObject> _panelHandle;
_panelHandle = await GameAssetService.Instance.LoadAssetAsync<GameObject>("UI.Login.Panel");
```

错误：

```csharp
var prefab = (await GameAssetService.Instance.LoadAssetAsync<GameObject>("UI.Login.Panel")).Asset;
```

### 2. UI `await` 返回后先判活

```csharp
var handle = await GameAssetService.Instance.LoadAssetAsync<GameObject>("UI.Login.Panel");

if (_isClosed || this == null || !gameObject)
{
    handle.Release();
    return;
}
```

### 3. 界面关闭时必须释放句柄

```csharp
_panelHandle?.Release();
_panelHandle = null;
```

### 4. `OnDestroy()` 必须兜底释放

所有动态加载资源的 UI，建议在 `OnDestroy()` 中再次调用释放逻辑。

### 5. 模块退出后建议回收内存

```csharp
await AssetBundleManager.Instance.UnloadUnusedAssetsAsync(true);
```

## 预热规则

### 1. 预热不等于业务已持有

例如：

- `PreloadGroupAsync("Login")` 是预热系统持有
- `LoadAssetAsync<GameObject>("UI.Login.Panel")` 是业务自己持有

这两份引用都要释放。

### 2. 释放预热组

```csharp
GameAssetService.Instance.ReleaseGroup("Login");
```

### 3. 预热适合阶段性模块

推荐：

- 登录模块
- 主城模块
- 战斗模块
- 场景切换前的公共资源

不建议所有资源长期常驻预热。

## 场景规则

### 1. 场景统一通过 key 加载

```csharp
await GameAssetService.Instance.LoadSceneAsync("Scene.MainCity");
```

### 2. Additive 场景必须主动卸载

```csharp
await GameAssetService.Instance.UnloadSceneAsync("Scene.Login", true, true);
```

## 常见错误

1. 只拿资源对象，不保存 `AssetHandle<T>`。
2. `await` 返回后不判活，直接实例化。
3. 界面关闭后没有 `Release()`。
4. 误以为“预热过了就不用释放业务 handle”。
5. 同一模块多次加载同一资源，但没有对每次加载结果分别释放。

## 一句话总结

- 谁加载，谁释放。
- 先保存 handle，再使用资源。
- UI 异步回来先判活。
- 预热引用和业务引用分开算。
- 业务层不干预底层共享加载。
