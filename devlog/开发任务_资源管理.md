# 资源管理开发任务

## 当前已实现功能

### 1. AssetBundle 运行时管理

- 已实现 `AssetBundleManager` 单例。
- 正式环境统一走 `AssetBundle` 异步加载。
- 支持 `AssetBundleManifest` 依赖解析与依赖自动加载。
- 支持 Windows、Android、iOS 平台路径处理。
- Android 运行时使用 `UnityWebRequestAssetBundle`。
- PC/iOS 运行时使用 `AssetBundle.LoadFromFileAsync`。
- 支持已加载 bundle 复用与引用计数管理。
- 支持 bundle 卸载与 `UnloadUnusedAssets` 内存回收。

### 2. 开发环境与正式环境切换

- 支持 `developmentMode` 开关。
- Unity Editor 开发模式下可直接从源资源路径加载。
- 正式环境 `AssetBundle` 模式下限制同步加载接口，避免同步/异步混用导致状态错乱。

### 3. 资源加载接口

- 支持 `AssetAddress` 资源地址定义。
- 支持 `LoadAssetAsync<T>()` 异步资源加载。
- 支持 `AssetHandle<T>` 句柄持有与释放。
- 同步加载接口仅保留给 Editor 开发模式调试使用。

### 4. 场景加载接口

- 支持 `SceneAddress` 场景地址定义。
- 支持 `BundleSceneLoader.LoadSceneAsync()` 场景异步加载。
- 支持场景卸载后自动释放对应 scene bundle。
- 支持 Editor 开发模式下直接加载源场景。

### 5. 预热系统

- 支持 bundle 级预热。
- 支持 asset 级预热。
- 支持分组预热 `PreloadGroupConfig`。
- 支持分组释放、bundle 释放、asset 释放。
- 支持预热进度回调。
- 支持预热资源缓存查询。

### 6. 分包下载扩展点

- 已抽象 `IBundlePackageService` 接口。
- 已提供 `LocalBundlePackageService` 本地直通实现。
- 后续可无侵入接入分包下载、校验和缓存管理。

### 7. 配置系统

- 已实现 `AssetAddressConfig`，统一维护资源 key 到地址映射。
- 已实现 `PreloadGroupConfig`，统一维护分组、分包、预热资源配置。
- 已提供 `AssetAddressConfigValidator` 校验工具。
- 已提供 `PreloadGroupConfigValidator` 校验工具。

### 8. 编辑器构建工具

- 已实现 `AssetBundleBuilder`。
- 支持按当前平台构建 AssetBundle。
- 支持 Android、iOS、Windows64 单独构建。
- 输出目录统一为 `Assets/StreamingAssets/AssetBundles/<Platform>/`。

### 9. 调试与稳定性

- 已实现统一错误码 `BundleErrorCode`。
- 已实现统一异常类型 `BundleException`。
- 已实现统一日志入口 `BundleLogger`。
- 已实现加载、预热、释放全链路日志。
- 已实现重复释放和非法释放保护。
- 已提供当前已加载 bundle、预热 bundle、预热 asset、预热 group 的调试查询接口。

### 10. 对外门面

- 已实现 `GameAssetService` 统一资源服务入口。
- 外部模块可通过该门面进行资源加载、场景加载、预热、释放和调试查询。

## 近期新增开发内容（2026-03-16 ~ 2026-03-17）

### 11. AssetBundle 构建规则调整

- AssetBundle 输出目录已调整为 `Assets/StreamingAssets/<Platform>/`。
- 运行时 bundle 文件名规则已统一为：`bundleName` 中的 `/` 替换为 `_`，并追加 `.bundle` 后缀。
- 主 manifest 保持使用平台名，例如 `StandaloneWindows64`。
- 运行时加载路径已与新的平铺输出规则联动，业务层仍可按逻辑 `bundleName` 使用，底层会统一转换为实际 bundle 文件名。
- 构建工具已移除按目录层级生成后再搬平的旧逻辑，改为直接生成目标文件名。

### 12. 自动设置 BundleName 工具增强

- `AssetBundleBuilder` 已新增自动设置 BundleName 工具。
- 工具会读取 `Assets/Env/BuildConfig.asset` 中的 `buildRootList`。
- 每个搜索根目录下的一级目录会被视为一个 bundle 分组单位。
- 会递归遍历一级目录下的 `Prefab` 与 `Scene` 资源，并统一设置对应的 `assetBundleName`。
- 当前生成的 `assetBundleName` 已直接使用运行时实际文件名格式，例如 `gameassets_ui_main.bundle`。

### 13. 构建流程增强

- 当前仅保留两个构建菜单：
- `MMORPG/Bundle/Build Current Target`
- `MMORPG/Bundle/Rebuild Current Target`
- `Build Current Target` 为增量构建。
- `Rebuild Current Target` 为清理输出目录后的全量重建。
- 构建完成后会自动清理平台目录下已经失效的旧 bundle 文件，避免历史产物残留。

### 14. 版本清单 (version.json)

- 平台目录下统一生成 `version.json`，不再生成 `bundle_manifest.json`。
- **update dll** 运行后会写入/更新 `dlls[]`（launch.dll.bytes、game.dll.bytes 的 md5/size）。
- **update res** 运行后会重建 `bundles[]`（基于当前 `.bundle` 文件），并保留已有 `dlls[]`。
- 清单包含：`platform`、`version`、`buildTime`、`dlls`、`bundles`。
- `dlls[]` 节点包含：`path`、`md5`、`size`。
- `bundles[]` 节点包含：`path`、`md5`、`size`、`group`。
- `md5` 用于后续热更新版本对比与文件变更判断。
- `size` 用于下载展示与资源校验辅助。
- `group` 当前先固定写死为 `Main`，后续再扩展成正式分包配置。
- `version` 当前固定写死为 `1.0.1`。
- `buildTime` 当前记录为 UTC ISO 8601 时间字符串。

### 15. BuildConfig 配置收口

- `BuildConfig` 已增加统一配置入口，后续资源构建与运行时相关配置统一向该配置资产收口。
- 当前 `BuildConfig` 已包含：
- `version`
- `editorDevelopmentMode`
- `buildRootList`
- 已增加 `BuildConfig.AssetPath` 常量，统一定位 `Assets/Env/BuildConfig.asset`。

### 16. 开发模式开关联动调整

- 运行时开发模式开关来源已从 `GameAssetService` 内部字段迁移到 `BuildConfig.editorDevelopmentMode`。
- `GameAssetService` 已删除自身 `_developmentMode` 配置。
- `GameAssetService` 启动时会读取 `BuildConfig`，并将结果传给 `AssetBundleManager.Configure()`。
- 规则调整为：
- `UNITY_EDITOR` 下且 `BuildConfig.editorDevelopmentMode == true` 时，走 Editor 源资源加载。
- 其他情况统一走 AssetBundle 依赖加载。
- `GameAssetService` 在 Editor 下若未手动拖入 `BuildConfig`，会自动尝试从 `Assets/Env/BuildConfig.asset` 加载。

### 17. 冗余配置清理

- `GameAssetService` 中已删除 `_bundleRootFolderName`。
- `AssetBundleManager` 中已删除无实际作用的 `bundleRootFolderName` 配置链路。
- 当前 bundle 根路径统一由 `BundlePathUtility` 按 `StreamingAssets/<Platform>/` 规则生成。
