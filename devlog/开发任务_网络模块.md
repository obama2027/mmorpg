# 网络模块开发任务

## 当前已实现功能

### 1. Socket 连接核心

- 已实现 `SocketConnection`，支持单连接的完整生命周期管理。
- 支持 TCP 和 UDP 两种协议。
- 支持 IPv4、IPv6、DualStack 三种地址族。
- 支持异步非阻塞连接，连接超时可配置。
- 连接状态机：`Disconnected` → `Connecting` → `Connected` → `Reconnecting` → `Closed`。

### 2. 非阻塞收发

- 独立收发线程，协议收发不阻塞主线程。
- 收发线程通过 `ConcurrentQueue` 与主线程安全通信。
- 发送线程使用 `ManualResetEventSlim` 信号量驱动，无数据时休眠不空转。
- `SocketManager.PollAllConnections` 在主线程 `Update` 中派发已收到的消息和状态事件。

### 3. 拆包粘包处理

- 已实现 `SocketPacketFramer`，处理基于 2 字节长度头的包帧协议。
- 包结构：`[2字节body长度(大端序)] [2字节opcode(大端序)] [N字节payload]`。
- 2 字节长度头与 Skynet gate/netpack 原生协议兼容。
- 支持粘包拆包，内部维护循环缓冲区自动累积和拆分。
- 单包 body 最大限制约 64KB（65533 字节）。

### 4. 断线重连

- 支持自动断线重连，可配置重连次数和间隔。
- 重连期间状态切换为 `Reconnecting`，通过状态回调通知业务层。
- 重连全部失败后回到 `Disconnected` 状态。
- 重连时自动等待旧 IO 线程退出，防止新旧线程并存。

### 5. 心跳保活

- 底层自动发送心跳请求包，间隔可配置。
- 自动检测接收超时，超时后触发断线重连。
- 心跳响应包由底层过滤，不派发到业务层。
- 心跳 opcode（请求/响应）与服务端约定，通过配置指定。

### 6. 内存安全

- 收发 payload 缓冲区使用 `System.Buffers.ArrayPool<byte>.Shared` 池化，减少 GC 压力。
- `SocketPacketFramer` 的 `TryReadPacket` 和 `EncodePacket` 均从 ArrayPool 租借缓冲区。
- `SocketManager.PollAllConnections` 派发完成后自动归还接收缓冲区。
- `SocketConnection.Dispose` 时自动清空并归还发送队列和接收队列中残留的缓冲区。
- 发送队列容量有上限（默认 1024），超限时丢弃并记录日志，防止内存无限增长。

### 7. Protobuf 协议层

- 已实现 `SocketOpcodeDispatcher`，支持按 opcode 注册强类型 Protobuf 消息处理器。
- Protobuf 反序列化使用 `CodedInputStream(payload, 0, payloadLength)` 精确读取池化缓冲区。
- 支持原始字节处理器注册（`RegisterRaw`）。

### 8. 多连接管理

- 已实现 `SocketManager` 单例，管理多个 `SocketConnection` 实例。
- 支持按 connectionId 创建、连接、断开、销毁连接。
- `PollAllConnections` 遍历前拷贝快照，回调中修改连接字典不会崩溃。

### 9. 线程安全

- 连接状态 `_state` 使用 `volatile` 保证跨线程可见性。
- 发送队列计数使用 `Interlocked` 原子操作。
- 心跳时间戳使用 `Interlocked.Exchange` 更新。
- 重连和 `Dispose` 时调用 `WaitIOThreadsExit()` 确保旧 IO 线程已退出。

### 10. 错误与日志

- 已实现 `SocketErrorCode` 统一错误码枚举。
- 已实现 `SocketModuleException` 统一异常类型。
- 已实现 `SocketLogger` 统一日志入口，支持 Info、Warn、Error 分级。
- 连接、断线、重连、发送、接收全链路均有日志输出。

### 11. 对外门面

- 已实现 `SocketService` 统一网络服务入口。
- 业务层通过该门面进行连接管理、协议发送、handler 注册和状态监听。
- 业务层不直接操作 `SocketConnection` 和 `SocketManager`。

## 后续建议优先开发功能

### P1

- 重连间隔增加指数退避策略，避免服务器被集中重连冲击。
- `SocketMainThreadDispatcher` 当前为死代码，评估保留或移除。
- 实际接入一个登录/大厅服务器验证全链路连通性。
- 网络状态变化回调补充错误信息（如 `SocketErrorCode`），方便业务层精确提示。

### P2

- 增加网络统计数据收集（RTT、发送/接收吞吐量、重连次数）。
- 增加通信加密/安全层（防篡改、防重放）。
- 增加协议版本号或握手校验，防止客户端与服务端版本不匹配。
- UDP 可靠传输支持（KCP 或自定义 ARQ）。
- 弱网模拟工具，辅助开发阶段测试断线重连和心跳超时。

### P3

- 协议压缩（LZ4 / Snappy）减少传输量。
- 流量控制与拥塞管理。
- 多连接分组管理与优先级调度。
- 网络状态运行时调试面板。
