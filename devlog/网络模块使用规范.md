# 网络模块使用规范

## 适用范围

适用于客户端业务层使用当前网络模块的所有场景：

- **Socket**：连接管理、协议收发、断线重连、消息处理
- **HTTP**：REST 接口调用（GET/POST 等）
- **Download**：资源/大文件下载（内存/文件流式）

## 核心原则

1. 业务层统一通过 `SocketService` 操作，不直接接触 `SocketConnection` 和 `SocketManager`。
2. 协议收发不阻塞主线程，底层通过独立收发线程 + `ConcurrentQueue` 异步通信。
3. 协议派发在主线程 `Update` 中执行，handler 内可安全操作 Unity API。
4. 心跳、重连等机制由底层自动处理，业务层只需配置参数和监听状态变化。
5. payload 缓冲区使用 `ArrayPool` 池化，`RegisterRawHandler` 收到的 `byte[]` 不可长期持有。

## 连接管理

### 1. 创建并连接

使用默认配置：

```csharp
int connId = SocketService.Instance.CreateAndConnect("192.168.1.100", 8888);
```

使用自定义配置：

```csharp
var config = SocketConnectionConfig.Default("192.168.1.100", 8888);
config.HeartbeatIntervalMs = 10000;
config.HeartbeatTimeoutMs = 30000;
config.HeartbeatRequestOpcode = 1;
config.HeartbeatResponseOpcode = 2;

int connId = SocketService.Instance.CreateAndConnect(config);
```

### 2. 分步创建和连接

```csharp
int connId = SocketService.Instance.CreateConnection(config);
// ... 做一些准备工作 ...
SocketService.Instance.Connect(connId);
```

### 3. 断开连接

```csharp
// 断开（如果配置了 AutoReconnect，底层会自动重连）
SocketService.Instance.Disconnect(connId);

// 销毁（释放资源，不会重连）
SocketService.Instance.DestroyConnection(connId);
```

### 4. 销毁所有连接

```csharp
SocketService.Instance.DestroyAll();
```

## 协议发送

### 1. 发送 Protobuf 消息

```csharp
var req = new LoginRequest { Username = "player1", Password = "xxx" };
SocketService.Instance.Send(connId, 1001, req);
```

### 2. 发送原始字节

```csharp
byte[] data = ...;
SocketService.Instance.SendRaw(connId, 1001, data);
```

### 3. 发送队列满时

发送队列有容量上限（默认 1024），网络卡顿导致队列满时，后续 `Send` 会被丢弃并打印警告日志。业务层不需要处理此情况，网络恢复后队列会自动消化。

## 协议接收

### 1. 注册 Protobuf 消息处理器

```csharp
SocketService.Instance.RegisterHandler<LoginResponse>(1002, OnLoginResponse);

private void OnLoginResponse(int connectionId, LoginResponse msg)
{
    // 在主线程执行，可安全操作 UI
    Debug.Log($"Login result: {msg.Result}");
}
```

### 2. 注册原始字节处理器

```csharp
SocketService.Instance.RegisterRawHandler(1003, OnRawData);

private void OnRawData(int connectionId, byte[] payload, int payloadLength)
{
    // payload 来自 ArrayPool，不要保存引用
    // 如需保留数据，必须自行拷贝
    byte[] copy = new byte[payloadLength];
    Buffer.BlockCopy(payload, 0, copy, 0, payloadLength);
}
```

### 3. 取消注册

```csharp
SocketService.Instance.UnregisterHandler(1002);
SocketService.Instance.UnregisterAllHandlers();
```

## 连接状态监听

### 1. 监听状态变化

```csharp
SocketService.Instance.OnConnectionStateChanged += OnStateChanged;

private void OnStateChanged(int connId, SocketConnectionState oldState, SocketConnectionState newState)
{
    switch (newState)
    {
        case SocketConnectionState.Connected:
            // 连接成功或重连成功
            break;
        case SocketConnectionState.Reconnecting:
            // 断线了，正在重连，显示提示 UI
            break;
        case SocketConnectionState.Disconnected:
            if (oldState == SocketConnectionState.Reconnecting)
            {
                // 重连失败，弹出"网络不可用"对话框
            }
            break;
    }
}
```

### 2. 主动查询状态

```csharp
var state = SocketService.Instance.GetConnectionState(connId);
```

### 3. 连接状态说明

| 状态 | 含义 |
|------|------|
| `Disconnected` | 未连接或重连全部失败 |
| `Connecting` | 正在建立连接 |
| `Connected` | 已连接，可以收发数据 |
| `Reconnecting` | 断线后正在自动重连 |
| `Closed` | 连接已销毁，不可复用 |

## 心跳配置

心跳由底层自动处理，业务层无感知。心跳响应包不会派发到 handler。

```csharp
var config = SocketConnectionConfig.Default("192.168.1.100", 8888);
config.HeartbeatIntervalMs = 10000;        // 每 10 秒发一次心跳，0 = 不启用
config.HeartbeatTimeoutMs = 30000;         // 30 秒没收到任何数据判定断线
config.HeartbeatRequestOpcode = 1;         // 与服务端约定的 Ping opcode
config.HeartbeatResponseOpcode = 2;        // 与服务端约定的 Pong opcode
```

## 配置参数一览

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `Host` | - | 服务器地址 |
| `Port` | - | 服务器端口 |
| `Protocol` | `Tcp` | 协议类型：`Tcp` / `Udp` |
| `AddressFamily` | `IPv4` | 地址族：`IPv4` / `IPv6` / `DualStack` |
| `ConnectTimeoutMs` | 5000 | 连接超时（ms） |
| `ReconnectIntervalMs` | 3000 | 重连间隔（ms） |
| `MaxReconnectAttempts` | 5 | 最大重连次数 |
| `SendBufferSize` | 65536 | 发送缓冲区大小 |
| `RecvBufferSize` | 65536 | 接收缓冲区大小 |
| `AutoReconnect` | true | 是否自动重连 |
| `HeartbeatIntervalMs` | 0 | 心跳间隔，0 = 不启用 |
| `HeartbeatTimeoutMs` | 0 | 心跳超时，0 = 不检测 |
| `HeartbeatRequestOpcode` | 0 | 心跳请求 opcode |
| `HeartbeatResponseOpcode` | 0 | 心跳响应 opcode |
| `SendQueueCapacity` | 1024 | 发送队列上限，0 = 无限制 |

## HTTP 请求

通过 `UnityHttpClient` 调用 REST 接口，支持 GET/POST、取消、重试和超时。

### 1. 何时用 GET / POST

| 用途 | 方法 | 示例 |
|------|------|------|
| 拉取文本/JSON | `GetStringAsync` | 公告、配置、排行榜 |
| 拉取二进制 | `GetBytesAsync` | 小资源、校验文件 |
| 提交 JSON body | `PostJsonAsync` | 登录、上报、写接口 |
| 提交表单 | `PostFormAsync` | 表单登录、文件上传 |
| 自定义 body | `PostAsync` | 任意 Content-Type |

```csharp
// GET 文本
string json = await UnityHttpClient.Instance.GetStringAsync("https://api.example.com/config");

// POST JSON
var resp = await UnityHttpClient.Instance.PostJsonAsync<LoginResponse>("https://api.example.com/login", new LoginRequest { ... });
```

### 2. 请求选项（超时、取消、重试）

```csharp
var options = new HttpRequestOptions
{
    TimeoutSeconds = 30,
    MaxRetries = 2,
    RetryDelayMs = 1000,
    CancellationToken = cts.Token  // 支持取消
};

string data = await UnityHttpClient.Instance.GetStringAsync(url, options);
```

### 3. 异常处理

- `HttpModuleException`：HTTP 错误（非 2xx、网络失败、超时等）
- `OperationCanceledException`：用户取消（`CancellationToken` 触发）

```csharp
try
{
    var data = await UnityHttpClient.Instance.GetStringAsync(url, options);
}
catch (OperationCanceledException)
{
    // 用户取消，无需重试
}
catch (HttpModuleException ex)
{
    // ex.ErrorCode, ex.Message, ex.StatusCode
}
```

### 4. 重试行为

失败时按 `MaxRetries` 和 `RetryDelayMs` 重试；取消不会重试。重试间隔为 `RetryDelayMs * (attempt + 1)`。

## 网络下载

通过 `NetworkDownloader` 下载资源，支持内存/文件两种模式，文件模式使用流式写盘，适合大文件。

### 1. 内存 vs 文件

| 模式 | 方法 | 适用场景 |
|------|------|----------|
| 内存 | `DownloadToMemoryAsync` | 小文件、需在内存中解析 |
| 流 | `DownloadToStreamAsync` | 需写入 `Stream` 的场景 |
| 文件 | `DownloadToFileAsync` | 大文件、资源包，流式写盘，不占用大内存 |

```csharp
// 小文件进内存
byte[] data = await NetworkDownloader.Instance.DownloadToMemoryAsync(url);

// 大文件直接写盘（流式，推荐）
await NetworkDownloader.Instance.DownloadToFileAsync(url, "D:/cache/bundle.asset", new DownloadOptions { UseTempFile = true });
```

### 2. 取消与进度

```csharp
var cts = new CancellationTokenSource();
var options = new DownloadOptions
{
    UseTempFile = true,
    Overwrite = true,
    CancellationToken = cts.Token,
    OnProgress = p => Debug.Log($"Progress: {p:P0}")  // 0~1
};

await NetworkDownloader.Instance.DownloadToFileAsync(url, destPath, options);

// 用户点击取消时
cts.Cancel();
```

### 3. 异常处理

- `DownloadModuleException`：下载失败、IO 错误等
- `OperationCanceledException`：用户取消

```csharp
try
{
    await NetworkDownloader.Instance.DownloadToFileAsync(url, path, options);
}
catch (OperationCanceledException) { /* 用户取消 */ }
catch (DownloadModuleException ex) { /* ex.ErrorCode, ex.Message */ }
```

### 4. 重试与临时文件

- 失败时按 `MaxRetries`、`RetryDelayMs` 重试；取消不重试。
- `UseTempFile = true` 时先写到 `path + ".tmp"`，成功后再替换为目标文件，避免下载中断留下残缺文件。

## 包结构

```
[2字节包体长度(大端序)] [2字节opcode(大端序)] [N字节protobuf payload]
```

- 包体长度 = 2（opcode）+ N（payload 字节数）
- 2 字节长度头与 Skynet gate/netpack 原生协议兼容
- 单包最大 body 大小：65533 字节（约 64KB）

## 常见错误

**Socket**

1. 直接操作 `SocketManager` 或 `SocketConnection`，绕过 `SocketService`。
2. 在 `RegisterRawHandler` 的回调中保存 `payload` 引用，该缓冲区会被归还到 ArrayPool。
3. 忘记监听 `OnConnectionStateChanged`，导致断线后 UI 无任何提示。
4. 在 `handler` 回调中做耗时操作，阻塞后续包的派发。
5. 服务器踢人场景下仍然让底层自动重连，应在收到踢人协议后调用 `DestroyConnection`。

**HTTP / Download**

6. 大文件用 `DownloadToMemoryAsync` 导致内存压力，应使用 `DownloadToFileAsync` 流式写盘。
7. 未捕获 `OperationCanceledException`，将取消与业务异常混在一起处理。
8. 下载大文件不设置 `OnProgress`，用户无法看到进度。

## 一句话总结

**Socket**

- 统一走 `SocketService`，不直接碰底层。
- Protobuf handler 在主线程派发，可安全操作 UI。
- Raw handler 收到的 `byte[]` 不可持有，需要时自行拷贝。
- 心跳和重连底层自动处理，业务层只监听状态变化。
- 服务器主动踢人时，业务层调用 `DestroyConnection` 阻止重连。

**HTTP / Download**

- 大文件用 `DownloadToFileAsync` 流式写盘，小文件用 `DownloadToMemoryAsync`。
- 通过 `CancellationToken` 支持取消，通过 `OnProgress` 报告下载进度。
- 异常区分 `OperationCanceledException`（取消）与 `HttpModuleException`/`DownloadModuleException`（业务错误）。
